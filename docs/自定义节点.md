# 自定义节点

## 概述

每个节点双击后会弹出控制面板，面板内容由节点的 `Property` 属性描述符自动生成。以下说明如何自定义节点的属性面板。

---

## 1. 基础的 Property 定义

在节点类中定义 `Property` 类属性即可自动出现在面板中：

```python
from core.node_base import OpenCVNodeDataBase, Property, PropertyGroupNames

class MyFilter(OpenCVNodeDataBase):
    __group__ = "滤波模块"

    # 基本参数
    ksize = Property(5, name="核大小", group=PropertyGroupNames.RUN_PARAMETERS,
                     description="卷积核的尺寸")
    threshold = Property(128, name="阈值", group=PropertyGroupNames.RUN_PARAMETERS,
                         min_val=0, max_val=255)

    # 只读结果参数
    result_count = Property(0, name="检测数量", group=PropertyGroupNames.RESULT_PARAMETERS,
                            readonly=True)

    def __init__(self):
        super().__init__()
        self.name = "自定义滤波"
```

### Property 参数说明

| 参数 | 类型 | 说明 |
|------|------|------|
| `default` | Any | 默认值 |
| `name` | str | 面板中显示的标签 |
| `group` | str | 分组名（决定在哪个标签页显示） |
| `description` | str | 鼠标悬停提示 |
| `readonly` | bool | 只读显示 |
| `order` | int | 同组内排序 |
| `editor` | str | 自定义编辑器类型（见第3节） |
| `choices` | list | 下拉选项列表 |
| `min_val` | number | 最小值约束 |
| `max_val` | number | 最大值约束 |
| `step` | float | 步进值 |
| `decimals` | int | 小数位数 |

### 内部分组名

```python
class PropertyGroupNames:
    RUN_PARAMETERS = "运行参数"      # 主标签页
    BASE_PARAMETERS = "基本参数"
    RESULT_PARAMETERS = "结果参数"
    FLOW_PARAMETERS = "流程控制"
    DISPLAY_PARAMETERS = "显示参数"
    OTHER_PARAMETERS = "其他参数"
```

## 5. 完整示例

```python
"""自定义高斯增强节点"""
import cv2
from core.node_base import OpenCVNodeDataBase, Property, PropertyGroupNames
from core.data_packet import FlowableResult

class EnhancedBlur(OpenCVNodeDataBase):
    __group__ = "滤波模块"

    # ── 运行参数 ──
    ksize = Property(5, name="核大小", group=PropertyGroupNames.RUN_PARAMETERS,
                     min_val=1, max_val=31, step=2)
    sigma = Property(1.0, name="Sigma", group=PropertyGroupNames.RUN_PARAMETERS,
                     min_val=0.0, max_val=10.0, step=0.1, decimals=1)
    mode = Property("blur", name="模糊模式", group=PropertyGroupNames.RUN_PARAMETERS,
                    choices=["blur", "median", "bilateral"])

    # ── 结果参数 ──
    elapsed_ms = Property(0.0, name="耗时(ms)", group=PropertyGroupNames.RESULT_PARAMETERS,
                          readonly=True)

    def __init__(self):
        super().__init__()
        self.name = "高斯增强"

    def invoke_core(self, src, from_node, diagram):
        import time
        t0 = time.time()
        k = self.ksize
        if k % 2 == 0:
            k += 1  # 确保核大小为奇数

        if self.mode == "blur":
            result = cv2.GaussianBlur(src, (k, k), self.sigma)
        elif self.mode == "median":
            result = cv2.medianBlur(src, k)
        else:
            result = cv2.bilateralFilter(src, k, self.sigma * 30, self.sigma * 10)

        self.elapsed_ms = round((time.time() - t0) * 1000, 1)
        return self.ok(result)
```

将此文件放到 `nodes/blurs/` 目录下，程序启动时自动发现并注册。

## 6. 面板生命周期

- **打开**：双击节点 → `open_node_dialog()` → 创建 `PropertyPanel` → 调用 `set_node(presenter)` → 读取 `get_property_descriptors()` → 生成表单
- **修改**：用户在面板中修改值 → 实时同步到节点对象（`property.fset`）
- **刷新**：面板每 500ms 自动调用 `_do_refresh()` 同步只读属性（如 `elapsed_ms`）
- **关闭**：点击"关闭"按钮 → 对话框销毁 → 节点保留当前属性值

## 7. 注册与发现

节点类只需要：
1. 继承 `OpenCVNodeDataBase`（或其它 `NodeBase` 子类）
2. 设置 `__group__` 类属性指定分组
3. 文件放在 `nodes/` 目录下

程序通过 `node_group.discover_module()` 自动扫描 `nodes/` 下所有模块并注册。

## 8. 常见问题

**Q: 修改 Property 默认值不生效？**
A: 在 `__init__` 中设置初始值时，用 `self.xxx = value` 而非直接赋值给类属性。

**Q: 面板值修改后节点没有反应？**
A: Property 描述符会自动触发变更通知。如果需要额外逻辑，在 `__init__` 后手动监听或在 `invoke_core` 中读取最新值。

**Q: 如何在面板中隐藏某个属性？**
A: 不将其定义为 `Property` 类属性，改为普通实例属性即可。

**Q: 如何让属性在特定条件下只读？**
A: 重写 `property_panel.py` 中 `PropertyPanel._is_readonly()` 方法，或使用 `editor_registry` 自定义编辑器。

## 9. 源节点架构与解耦

### 9.1 架构概览

所有图像源节点（"图像文件"、"刀架测试图"、"摄像头" 等）共享同一套基础设施，新增源节点**无需编写任何 UI 代码**。

```
│                   基础设施（已内置）                │
├──────────────────────────────────────────────────┤
│  SrcFilesVisionNodeData（core/node_base.py）       │
│    ├─ src_file_paths : list[str]   文件列表        │
│    ├─ src_file_path : str          当前选中文件     │
│    ├─ add_files_from_folder(dir)   扫描文件夹      │
│    ├─ clear_files() / move_next()  文件操作        │
│    └─ load_default()               加载默认数据     │
├──────────────────────────────────────────────────┤
│  FlowResourcePanel（gui/flow_resource_panel.py）    │
│    ├─ set_node(SrcFilesVisionNodeData)             │
```

### 9.2 如何新增一个图像源节点

只需定义一个子类，设置搜索关键词和中文名，**不需要写任何 UI 代码**：

```python
# nodes/sources/zoo_sources.py

class _ZooSrcImageBase(SrcFilesVisionNodeData, OpenCVNodeDataBase):
    """Zoo 图像源基类 —— 所有系统测试图都继承此类。"""
    __group__ = "系统数据源"
    _path_filter: str = ""       # 文件名过滤关键词

    def __init__(self):
        SrcFilesVisionNodeData.__init__(self)
        OpenCVNodeDataBase.__init__(self)
        self.load_default()      # 从 assets/images 加载匹配图片

    def load_default(self):
        super().load_default()   # 加载所有 assets/images 下的图片
        if self._path_filter:
            self.src_file_paths = [p for p in self.src_file_paths
                                    if self._path_filter.lower() in p.lower()]
            if self.src_file_paths:
                self.src_file_path = self.src_file_paths[0]

    def invoke_core(self, src, from_node, diagram):
        path = self.src_file_path
        mat = cv2.imread(path, cv2.IMREAD_COLOR)
        self.pixel_width, self.pixel_height = mat.shape[1], mat.shape[0]
        return self.ok(mat, f"已加载: {os.path.basename(path)}")

    def _update_result_image_source(self):
        self._result_image_source = self._mat
```

### 9.3 数据流

```
节点创建
  __init__() → load_default() → add_files_from_folder("assets/images")
  → 过滤 _path_filter → src_file_paths 列表 → 选第一张为 src_file_path

节点被选中（单击）
  MainWindow._select_node() → 检测到 SrcFilesVisionNodeData
  → FlowResourcePanel.set_node() → 异步加载缩略图 → 显示缩略图条
  → ImageViewer 显示当前图片

双击节点 → 属性面板
  "运行参数" 标签页 → src_file_path 属性
  → 属性面板检测到 SrcFilesVisionNodeData 子类
  → 浏览按钮打开文件夹选择器（不是文件选择器）
  → 选择文件夹 → add_files_from_folder() → 刷新缩略图

点击缩略图
  FlowResourcePanel._on_thumbnail_clicked()
  → node.src_file_path = 选中文件
  → 发射 file_selected 信号 → MainWindow 刷新图像区
```

### 9.4 解耦关键点

| 组件 | 职责 | 对子类的要求 |
|------|------|-------------|
| `SrcFilesVisionNodeData` | 文件列表管理、加载、切换 | 无需额外代码 |
| `FlowResourcePanel` | 缩略图显示、点击切换 | 自动检测 `src_file_paths` |
| `PropertyPanel` | 文件夹选择器、属性编辑 | 自动检测节点类型 |

## 10. 完整注册流程：添加 "ORB 特征匹配" 节点

以下逐步演示如何在"模板匹配模块"中添加一个 ORB 特征匹配节点。

### 前置知识

ORB (Oriented FAST and Rotated BRIEF) 是 OpenCV 内置的免费特征检测算法（无需 `opencv_contrib`），流程为：

```
上游节点输出图像(scene)
        ↓
  ORB 检测 → 关键点 + 描述符
        ↓
  与模板图像的关键点匹配(BFMatcher)
        ↓
  过滤优秀匹配点 (Lowe's ratio test)
        ↓
  绘制匹配结果 → 输出
```

### 步骤 1：确定节点文件位置

查看 `main.py` 第 73 行，"模板匹配模块"对应包名 `nodes.template_matchings`：

```python
# main.py

```

所以文件路径为：**`nodes/template_matchings/`** 目录。可以直接在现有 `template_matching.py` 末尾添加，或新建文件如 `orb_matching.py`。

> **解耦原则**：新建独立文件更干净。
> `main.py` 的 `_discover_nodes()` 会自动扫描该目录下所有 `.py` 文件中的 `NodeBase` 子类。

```python
class OrbFeatureMatchingNode(VisionNodeData, OpenCVNodeDataBase):
    """ORB 特征匹配节点——检测并匹配两图之间的 ORB 特征点。

    输入：上游节点输出的图像（scene） + 用户设置的模板图像
    输出：绘制了匹配连线后的结果图像
    """
    __group__ = "模板匹配模块"          # ← 决定出现在哪个工具箱分组

    # ── 运行参数（用户可调）──
    n_features = Property(500, name="特征点数量",
                          group=PropertyGroupNames.RUN_PARAMETERS,
                          description="ORB 检测的最大特征点数量",
                          min_val=100, max_val=5000, step=100)

    ratio_threshold = Property(0.75, name="Lowe比值阈值",
                               group=PropertyGroupNames.RUN_PARAMETERS,
                               description="最近邻距离与次近邻距离的比值上限",
                               min_val=0.5, max_val=0.95, step=0.05, decimals=2)

    draw_matches = Property(True, name="绘制匹配线",
                            group=PropertyGroupNames.RUN_PARAMETERS,
                            description="是否在输出图像上绘制匹配连线")

    # ── 结果参数（只读）──
    match_count = Property(0, name="匹配数量",
                           group=PropertyGroupNames.RESULT_PARAMETERS,
                           readonly=True)

    good_match_count = Property(0, name="优秀匹配数",
                               group=PropertyGroupNames.RESULT_PARAMETERS,
                               readonly=True)
```

### 步骤 6：实现核心算法 `invoke_core`

```python
    def invoke_core(self, src_image_node_data, from_node_data, diagram) -> FlowableResult:
        # 1. 获取输入图像（上游节点输出）
        mat = from_node_data.mat if from_node_data else None
        if mat is None:
            return self.error(None, "无输入图像，请连接上游节点")

        # 2. 获取模板图像
        template = self._template_image
        if template is None:
            return self.error(mat, "未设置模板图片，请在属性面板中设置")

        # 3. 创建 ORB 检测器并提取特征
        orb = cv2.ORB_create(nfeatures=self.n_features)
        kp1, des1 = orb.detectAndCompute(template, None)
        kp2, des2 = orb.detectAndCompute(mat, None)

        if des1 is None or des2 is None or len(des1) == 0 or len(des2) == 0:
            return self.ok(mat, "无法提取足够特征点")

        # 4. BFMatcher 暴力匹配
        bf = cv2.BFMatcher(cv2.NORM_HAMMING, crossCheck=False)
        matches = bf.knnMatch(des1, des2, k=2)

        # 5. Lowe's ratio test —— 过滤优秀匹配
        good = [m for m, n in matches if m.distance < self.ratio_threshold * n.distance]

        # 6. 绘制结果
        out = mat.copy()
        if self.draw_matches:
            out = cv2.drawMatches(template, kp1, mat, kp2, good, out,
                                   flags=cv2.DrawMatchesFlags_NOT_DRAW_SINGLE_POINTS)

        self.match_count = len(matches)
        self.good_match_count = len(good)

        return self.ok(out, f"ORB 匹配: {len(good)}/{len(matches)} 个特征点")

    def _update_result_image_source(self):
        self._result_image_source = self._mat
```

### 步骤 7：注册到发现系统

`main.py` 的 `_discover_nodes()` 会自动扫描 `nodes/template_matchings/` 下的所有 `NodeBase` 子类。只要类设置了 `__group__ = "模板匹配模块"`，就能被自动发现并注册。

**无需手动修改 `main.py`。** 这是解耦的关键——新增节点文件放在对应目录即可。

```python
        return self._template_image

    def invoke_core(self, src_image_node_data, from_node_data, diagram) -> FlowableResult:
        mat = from_node_data.mat if from_node_data else None
        if mat is None:
            return self.error(None, "无输入图像，请连接上游节点")

        template = self._template_image
        if template is None:
            return self.error(mat, "未设置模板图片")

        orb = cv2.ORB_create(nfeatures=self.n_features)
        kp1, des1 = orb.detectAndCompute(template, None)
        kp2, des2 = orb.detectAndCompute(mat, None)

        if des1 is None or des2 is None or len(des1) == 0 or len(des2) == 0:
            return self.ok(mat, "无法提取足够特征点")

        bf = cv2.BFMatcher(cv2.NORM_HAMMING, crossCheck=False)
        matches = bf.knnMatch(des1, des2, k=2)
        good = [m for m, n in matches if m.distance < self.ratio_threshold * n.distance]

        out = mat.copy()
        if self.draw_matches:
            out = cv2.drawMatches(template, kp1, mat, kp2, good, out,
                                   flags=cv2.DrawMatchesFlags_NOT_DRAW_SINGLE_POINTS)

        self.match_count = len(matches)
        self.good_match_count = len(good)
        return self.ok(out, f"ORB 匹配: {len(good)}/{len(matches)} 个特征点")

    def _update_result_image_source(self):
        self._result_image_source = self._mat
```
